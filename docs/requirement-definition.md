# もちログ

介護施設での持ち物確認を、もっとかんたんに。

# 誰のどんな課題を解決するのか

ショートステイは、介護が必要な方が短期間だけ施設に宿泊し、介護や日常生活の支援を受けるサービスです。
利用者が日ごとに入れ替わり、衣類・日用品・薬・貴重品など多くの持ち物を持参するため、介護職員は入所時・退所時に持ち物確認を行う必要があります。

現場では、紙のチェック表や口頭確認で対応しているところも多く、「チェック柄の服」と書いても実際にどの服なのか伝わりにくいといった問題があります。
また、不足品が見つかった際には、家族への連絡や施設貸与品への切り替えといった対応が必要になりますが、忙しい業務の中では対応漏れや引き継ぎ漏れが起こりやすい状況があります。

# なぜそれを解決したいのか

私の母は介護施設で働いています。職場は忙しく、人手不足の中で多くの業務を回しており、肉体的にも精神的にもつらそうな状況がありました。
「仕事が大変で辞めたい」と口にしていたこともあり、少しでも力になれることがないかと感じていました。
持ち物確認のように毎日のように発生する業務を少しでも楽にできれば、母のような職員の負担を減らせるのではないかと思い、このテーマを選びました。

# どうやって解決するのか

持ち物確認業務をWebアプリとして電子化することで解決します。

- 写真と一緒に持ち物を記録できるため、職員間で「どの服か」を具体的に共有しやすくなります
- 同じ利用者が繰り返し利用することが多いショートステイの特性を活かし、前回の持ち物一覧を初期表示することで、毎回一から入力する手間を省けます
- 日用品のサービス利用有無と実際の持参状況のずれをアラートで通知し、対応漏れを早く発見できるようにします
- 不足品への対応状況（未対応・家族連絡済み・施設貸与・解決済み）を記録・共有できるようにすることで、引き継ぎをスムーズにします

# 機能要件

## MVP
- 事業所ごとの共通アカウントによるログイン
- 利用者情報の登録
- 利用者ごとの持ち物一覧の登録
- 前回の持ち物一覧を元にした初期表示
- 持ち物ごとの写真登録
- 入所時/退所時の持ち物チェック

## 追加機能（スコープ外）
- 日用品ごとのサービス利用有無の登録
- サービス未利用かつ未持参の場合のアラート表示
- 日用品不足時の状態管理（未対応、家族連絡済み、施設貸与、解決済み）
- 印刷用画面の表示

# 非機能要件
- 紙業務に慣れた職員でも迷わず使えるシンプルなUI
- PC・タブレット・スマートフォンでのレスポンシブ対応
- GitHubへプッシュ時に静的解析で自動チェック
- GitHubのmainブランチにマージしたら自動デプロイ
